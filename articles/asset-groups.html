Asset Groups

📅 Published June 2025 📂 Platform: Keytelemetrics 👤 Audience: Administrators

Asset Groups were designed to control and manage assets of the same type or group. Assets can be assigned to multiple groups, allowing for comparison of information across groups. There are no restrictions per user — a user can see any group's assets.

Applies to: Assets  ·  Alerts  ·  Reports

1. Understanding the Structure

Asset Groups use a parent-child relationship. Unlike Cost Centres, there can be multiple parent Asset Groups. The highest level is called Root. Children can be added under any parent, and children may themselves act as parents.

By default, one parent Asset Group exists called Default. This can be renamed, and children created underneath it.

Tip: Plan your Asset Group structure on paper before creating it in the platform. Consider how you want to group vehicles — by type, depot, region, or fleet category.

2. Creating Asset Groups

Step 1 — Rename the Default Group

Navigate to Administration → Organisation → Asset Groups. Select the Default group, click Edit next to Asset Group Details, and rename it. Keep the Parent set to Root, then Save.

Step 2 — Add Child Groups

Click the + (Plus) button to create a new Asset Group. Select the renamed parent (or Root) and enter a name. Keep adding groups until your structure is complete.

Example structure: All Assets → Vehicles → LDV / Personal Cars  |  All Assets → Personnel

3. Editing Asset Group Information

You can edit any Asset Group to change its Name and Parent, and to add Contacts. Contacts added to a group will be visible to users on the Overview and Alerts sections of the platform with clickable links.

4. Viewing Assets in a Group

Select any Asset Group from the list, then choose the Assets section in the right panel to see all assets assigned to that group, along with their group membership details.
