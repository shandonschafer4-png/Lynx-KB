User Roles

📅 Published June 2025 📂 Platform: Keytelemetrics 👤 Audience: Administrators

Roles control what a user can see and do on the platform. They define access rights at a granular level — for example, allowing a user to list assets but not create or edit them. Roles can be created at both Vendor and Client level.

Default: The only built-in role is Administrator, which grants full view and edit access to everything the user has visibility of.

1. Creating a Role

Navigate to Administration → Accounts → Roles, then click the + button and complete the Role Wizard.

StepAction
1Click + to open the User Role Wizard
2Enter a Name and optional Description for the role
3Click Next
4Select the access rights (see table below)
5Click Save

2. Available Access Rights

Each permission area can be granted List and/or Create access. Some areas have additional granular rights.

Tip: Create roles that match your team's job functions — e.g. a "Fleet Viewer" role with List only, a "Fleet Manager" role with List + Create, and a "Technician" role with device-specific rights.

3. Assigning a Role to a User

A Role can be assigned when creating a new user, or by editing an existing user. Roles can also be assigned in bulk.

Individual User

Navigate to Administration → Users, select the user, click Edit on the User Roles section, and select the appropriate role.

During User Creation

After completing the personal details step in the User Wizard, the next screen presents the Role selection. Choose the applicable role before finishing.

Note: If no custom roles have been created yet, only Administrator will appear in the role selection list.
