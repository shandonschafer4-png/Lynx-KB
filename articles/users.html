Users

📅 Published June 2025 📂 Platform: Keytelemetrics 👤 Audience: Administrators

Users access the platform and — depending on the Features enabled on the Client and what Roles they have been assigned — can view and manage items. Users can be created at Vendor level (accessing all clients) or at Client level (accessing only their specific client).

1. User Creation Methods

MethodHow It WorksRecommended?
INVITE User receives an email with an activation link and sets their own password Yes — preferred method
AUTO Password is auto-generated by the platform. User must use the Forgot? link or an admin resets the account Acceptable if no custom domain
Important: The invite method requires a Custom Domain to be configured at Vendor level. Without this, the activation link will not generate correctly.

2. Creating a User

Navigate to Administration → Accounts → Users, then click the + button and complete the User Wizard.

FieldDescription
NameFull name of the user
Email AddressUsed for login and notifications
Mobile NumberUsed for SMS alert notifications
Cost CentreAssigns the user to a Cost Centre, controlling their asset visibility
Time ZoneEnsures trip and alert times display correctly for the user's location
LanguagePlatform display language for the user
Send InviteWhen checked, sends an activation email. Uncheck to auto-generate a password instead

After completing personal details, click Next to select the user's Role. If no custom roles have been created, only Administrator will appear.

3. Resetting a User's Account

If a user has forgotten their password and cannot use the login screen to reset it, an administrator can reset the account. This suspends the user account and sends them an email with a link to create a new password.

StepAction
1Select the user in Administration → Users
2Click the Reset button in the user detail panel
3Click OK to confirm — the user will receive a reset email
Note: This reset option also works for accounts where a password was auto-generated during creation and the user never set their own.

4. Configuring User Notifications

To receive alert notifications, edit the Notifications section on the user's profile. Configure the following:

SettingDescription
Allow SMS From / ToTime window during which SMS notifications can be sent to this user
Low PriorityNotification method: None, Email, SMS, or Email & SMS
Medium PriorityNotification method: None, Email, SMS, or Email & SMS
High PriorityNotification method: None, Email, SMS, or Email & SMS
Prerequisite: An Email Server and/or SMS Gateway must be configured at the vendor level before email or SMS notifications will work.
